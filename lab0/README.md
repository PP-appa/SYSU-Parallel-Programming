# 并行程序设计与算法 - Lab 0

本仓库为“并行程序设计与算法”课程的 Lab 0 实验代码与报告。

实验主题为：**环境配置与串行矩阵乘法**。主要通过实现基础的密集矩阵乘法（$1024 \times 1024$ 方阵），探索并对比不同层次优化手段（如内存访问局部性、编译优化、指令级并行度以及工业级数学库 Intel MKL）带来的性能差异。

## 目录结构

```text
lab0/
├── src/                    # 源码存放目录
│   ├── gemm_basic.py       # Python 原生实现的矩阵乘法（基准性能下限）
│   ├── gemm_basic.cpp      # C/C++ 三层循环的基础矩阵乘法
│   ├── gemm_ikj.cpp        # 循环顺序重排优化（提高空间局部性和 Cache 命中率）
│   ├── gemm_unroll.cpp     # 循环展开优化（减少跳转，增加 Instruction-Level Parallelism）
│   └── gemm_mkl.cpp        # 调用 Intel MKL 并行数学核心库的工业级实现
├── run_and_save.py         # 自动化评测与正确性验证脚本
├── times.json              # 脚本运行生成的不同版本耗时记录表
├── report.md               # 实验分析结题报告
└── .gitignore              # Git 忽略配置
```

## 环境依赖

* **操作系统:** Linux (推荐 Ubuntu 20.04/22.04) 或 WSL (Windows Subsystem for Linux)
* **编译器:** GCC/G++，用于编译 C/C++ 源码
* **环境库:** Intel MKL (`sudo apt-get install intel-mkl`)
* **解释器:** Python 3 (用于运行性能分析脚本)

## 运行方法

如果环境中已经配置好 g++ 和 Intel MKL，可以直接运行项目根目录下的测试脚本，该脚本会自动：
1. 创建 `bin` 目录
2. 逐一编译各优化版本的 C++ 源码并携带正确的链接参数（包括 `-O3` 编译层级应用、链接 `-lmkl_rt` 等）
3. 顺序执行所有版本（包括 Python 版本）
4. 输出性能基准时间到终端及 `times.json`。

**执行命令：**
```bash
python3 run_and_save.py
```

如果在执行中报出缺少 MKL 头文件的错误，请确认 MKL 的安装路径是否已经在环境变量或是 `/usr/include/mkl` 等标准搜索路径下。

## 核心结论

在本实验针对 $1024 \times 1024$ 的单精度/双精度矩阵惩罚测试中，从低效的基础原生实现层层打磨，最终的测速体现出了巨大的量级差异：

- **基础实现层面**：直接从 Python 切换至使用 C 语言基础数组和指针（未开优化），即可取得接近 14 倍的单核提升。
- **算法架构层面**：仅仅通过将三层循环体从传统的 `i-j-k` 优化至具有空间局部存取特性的 `i-k-j` 模式，即可取得约 3 倍的额外加速比。
- **指令底层层面**：配合 `-O3` 指令优化和循环的手动展开（降低循环控制指令阻碍流水线发射），耗时降至原生 C 语言的 $\frac{1}{30}$ 左右。
- **工业库封装**：最终接入专为 CPU 硬件底层做过汇编级重构封装的 MKL 函数库，完成一次乘法仅需不到 0.04 秒（相对于最低下限 Python 版本完成同样的乘法取得了快将近 **3000 倍**的绝对加速比）。
